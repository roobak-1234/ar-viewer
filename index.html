Project Report: Multi-User AR Heart App

An Interactive Exploration of a Collaborative AR Application

Executive Summary

This report outlines the development of a collaborative, multi-user Augmented Reality (AR) application for visualizing and manipulating a complex 3D anatomical model of a human heart. The primary goal was to create a shared, interactive learning environment on standard mobile devices that operates in a relative, synchronized space. The application successfully implements robust session management, real-time object manipulation, and synchronized separation of individual anatomical parts. Significant challenges, particularly concerning network race conditions, were overcome to produce a stable and functional prototype.

Model Manipulation

The host has full control over the heart model using intuitive touch gestures. All manipulations are synchronized in real-time to all connected clients, ensuring everyone sees the same state.

  • One-Finger Drag: Moves the model in AR space.
  • Two-Finger Pinch: Scales the model larger or smaller.
  • Two-Finger Twist: Rotates the model with full 360° freedom.

Synchronized Part Separation

A key feature is the ability for the host to select and individually manipulate any of the model's eight anatomical subdivisions. This complex interaction is fully networked, allowing all users to see the separation in real-time.

Key Challenges & Solutions

Development involved overcoming significant technical hurdles common in multi-user AR applications. Click on each challenge below to learn about the problem and its robust solution.

Problem:

The most persistent issue was a "race condition" where manipulating a separated heart part would cause it to jitter, overlap, or snap back to its original position. This was caused by a conflict between the local input script (`HeartManipulator`) and the network synchronization script (`NetworkedHeartPart`) fighting for control over the object's transform.

Solution:

The final architecture resolved this by establishing a clear "chain of command." The `HeartManipulator` was designated as the **controller**, responsible only for detecting input and sending commands. The `NetworkedHeartPart` was designated as the **authoritative state manager**, responsible for updating its own `NetworkVariable`. This eliminated the conflict and resulted in smooth, stable manipulation.

Problem:

On several occasions, the Unity Editor continued to report compiler errors even after the code was corrected. This was traced to corrupted or outdated cached data from previously installed packages, especially after switching package versions multiple times.

Solution:

A "deep clean" of the project environment was performed by deleting the `Library` and `Packages` folders. This forced Unity to re-download all dependencies and rebuild the project library from a clean state, which definitively resolved these stubborn environmental issues.

Conclusion & Future Work

This project successfully demonstrates the creation of a feature-complete, multi-user collaborative AR application. It provides a stable platform for a shared experience where complex object manipulations are synchronized effectively in real-time. The final architecture for handling networked object manipulation is robust and scalable. Several exciting enhancements could be implemented in the future:

  • ✓
    True World Synchronization: Integrating a system like ARCore Cloud Anchors to allow for a physically co-located experience.
  • ✓
    Permissions System: Implementing a UI for the host to grant manipulation control to other users.
  • ✓
    Anatomical Animations: Including pre-made animations (e.g., a beating heart) that can be triggered by the host.
  • ✓
    Persistent Sessions: Enhancing session management to allow for "persistent" lobbies where objects remain after rejoining.
  • ✓
    Model Annotation: Allowing users to place persistent labels or drawings in the shared AR space.
  • ✓
    UI/UX Refinements: Adding more intuitive visual feedback for selected parts and network status.
